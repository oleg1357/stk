Common Troubleshooting

We have identified some common errors a user may encounter while using our services.

Transaction Error When Using Decimals for Staking or Voting - 

If you are receiving a transaction error when attempting to create a new stake, remove an existing stake or vote, ensure that you are not using decimals in your transaction. Staking and voting systems will reject any transaction that is not a whole number of tokens. For example, if a users wallet balance is 100,000.19 STKHB they make stake or vote up to 100,000 or any whole number between 0 and 100,000. If a transaction number contains "." it will be rejected. Users may send, receive and trade fractions of tokens. 


Contract Information is Not Populating - 

If area's that should contain information about the contract (I.E. Total Supply, User Balance, Rewards, etc.) are not loading, blank or otherwise displays text errors instead of numbers, a user should check on a few different things. 

1 - Do you have a Web3 wallet provider installed in your current browser? (I.E Metamask) -

A Web3 wallet connection is required to populate certain contract data as well as view other contract information. Without a Web3 wallet provider connected, a user is unable to interact with or otherwise view the majority of the contract on the web interface. 

2 - Is your Web3 wallet provider connected to StakeHub's services? -

If your wallet is connected but contract information is not displaying correctly, refreshing the web page fixes this in most instances. If data still does not display correctly, check step 3.

3 - Is your Web3 wallet provider connected to the Binance Smart Chain? -

Ensure that your Web3 wallet is connected to the Binance Smart Chain. If you are connected to another network, contract data can not be populated as our contract is not deployed on any other network at this time. 

4 - Still not working? - 

Please Contact Us.


Vote Transactions are Being Rejected or Throwing Errors -

If a users vote transactions are being rejected there are a few causes. 

1 - Is your vote submission within the voting block end? -

If a voting block has ended all incoming votes will be rejected.

2 - Does your vote submission contain a decimal? -

All burn voting transactions that contains decimals will be rejected. Please use whole token numbers when voting. 

3 - Are you a current stake holder? -

For non-burn voting, your vote amount is the amount you currently have staked. If you do not currently have a stake, your vote will be rejected as it would count as 0 and lock you from voting until the next voting period.

4 - Have you already voted? -

For non-burn voting, you may only vote once per voting block. Ensure you have all your tokens staked you wish to vote with as you may not submit a new vote until a new voting block starts. 

5 - Still not working? -

Please Contact Us.


Rewards Pending is Not Updating - 

Depending on the size of your stake, it may take some time for your cumulative rewards to equal .001 or more. StakeHub's interface only display up to three decimal places. (I.E The current reward block time is 30 seconds. The current staking return APY is 30%. Rewards are credited to your pending balance 2880 times per day at a rate of .000000285388 % per 30 seconds. For every 1 token staked you earn ~ .000000285388 STKHB per 30 seconds.


Pending Rewards Reset to 0 After Removing or Creating New Stake - 

Pending rewards are calculated and distributed to the stake holder whenever a new stake is created or any portion of an existing stake is removed. This is built into the contract's code. You will be credited your full pending rewards balance when making any changes to an existing stake.
